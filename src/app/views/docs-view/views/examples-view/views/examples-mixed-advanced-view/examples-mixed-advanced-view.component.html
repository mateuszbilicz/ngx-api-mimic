This example contains controller class that mocks data.

Interfaces used in this example

Strong typing facilitates clarity. This interface setup expands on primitive shapes to introduce constraints like enumerations (UserRoles) and complex objects (Date).

Custom pipe

In ngx-api-mimic, Custom Pipes deliver an elegant mechanism to intercept and validate requests. By applying PipeTransform upon incoming definitions, your backend mock ensures structural integrity (for instance checking password strength) and raises unified NgxApiMimicException errors prior to function execution.

Custom guard

Guards natively restrict operation execution. MixedAdvanced_AuthGuard reads the authorization context cleanly from the execution loop and governs resource access based on the headers transmitted in the HTTP lifecycle.

Class code

Bringing data seeding together with our advanced rules, {{ '@UsingSchema' }} handles sophisticated types such as randomized dates (customFn) or specific enums. Coupled with declarative configurations like {{ '@UseGuards' }} or argument-level validators ({{ '@Body(BodyPasswordPipe)' }}), the class strictly ensures complete security and structural conformity.

Router and interceptor

We export the fully bound router as an interceptor bundle ready to intercept application wide asynchronous operations via standard HTTP calls.

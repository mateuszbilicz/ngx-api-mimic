This example contains controller class that mocks data.

Interfaces used in this example

Custom pipe

You can create custom pipes that can modify and validate variables inside endpoint params.

Custom guard

In this example, we implemented MixedAdvanced_AuthGuard guard which checks, if token inside Authorization header is equal to "test".

Class code

In this class we're using {{ '@Controller' }} and {{ '@UsingSchema' }} decodators to initialize controller and define mock schema. Also, we've added {{ '@UseGuards' }} decorator to implement guards. Inside POST /users (userCreate) we've added custom pipe BodyPasswordPipe to {{ '@Body' }} decorator. Inside the class we're defining specific methods via functions and method decorators.

Router and interceptor

We need to build router and create interceptor for our mock. router.usePrefix is defined to distinct different example API routes.

This advanced API mock example shows how to use Angular dependency injection to apply multiple guards (AuthGuard, AdminGuard), check headers, validate payload data directly from the body and return accurate HTTP exception statuses such as 401 or 403 securely within a full CRUD Controller.

Interfaces used in this example

Establishing distinct DTO (Data Transfer Object) models alongside strict enums solidifies our foundation. It ensures comprehensive type coverage when interacting directly with HTTP request payloads.

Guards

Secure execution requires strict verification boundaries. ApiMockAdvanced_AuthGuard reads incoming tokens from the simulated Authorization header and injects valid roles downstream. Conversely, ApiMockAdvanced_AdminGuard executes targeted role extraction, throwing 403 Forbidden errors where necessary to defend sensitive routes.

Class code

Constructing a comprehensive CRUD handler, the Controller utilizes a suite of structural decorators ({{ '@Get' }}, {{ '@Post' }}, {{ '@Put' }}, {{ '@Delete' }}). Observe how {{ '@UseGuards' }} operates smoothly at both controller scope for encompassing authentication, and explicitly at the method level (e.g. deleteTask) to enforce precise authorization scopes.

Router and interceptor

Finally, ngxApiMimicRouterFactory merges our logic handlers into an execution pipeline, ready to be dispatched through Angular's module providers.

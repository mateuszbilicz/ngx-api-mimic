This example contains controller class that mocks data.

Interfaces used in this example

It is a best practice to organize endpoint payload definitions using strict TypeScript interfaces. Here, a baseline schema models our entities logically.

Class code

The foundation of our mocked endpoint lies within the class decorated with {{ '@Controller' }}. By applying the {{ '@UsingSchema' }} decorator, we automatically generate a fully populated structure based on the provided data shapes (filling 100 entries automatically using our built-in generators). Inside this class, HTTP handling strategies are elegantly mapped via annotations like {{ '@Get' }}, {{ '@Post' }}, {{ '@Put' }}, and {{ '@Delete' }}, constructing comprehensive CRUD capabilities seamlessly.

Router and interceptor

To tie these simulated routes directly into our HttpClient infrastructure, we instantiate a mock router namespace via ngxApiMimicRouterFactory. The usePrefix function separates context blocks allowing multiple API mocks to co-exist without overriding each other.

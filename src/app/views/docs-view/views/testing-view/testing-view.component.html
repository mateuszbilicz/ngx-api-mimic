The standard approach for testing your application when using the ngx-api-mimic library. It integrates seamlessly with testing runners like Jest, Vitest, or Karma inside Angular 16+ environments.

Mock Data Isolation

When writing unit tests for services connected directly to NgxApiMimicMockData or using the @UsingSchema decorator, ensure you disable localStorage persistence to prevent state leaks across different test cases. You can do this by setting the preventStore flag to `true` (e.g. passing true as the 3rd argument to NgxApiMimicMockData).

API Mock with Angular HttpClient

Testing services that invoke HTTP calls normally requires HttpTestingController. However, when testing components or services against your fake endpoints, you can completely sidestep HttpTestingController and just inject the factory ngxApiMockInterceptorFactory right into your TestBed. By doing so, your Angular HttpClient calls will pass through the mock infrastructure transparently and quickly. Be sure to configure delays to 0 for synchronous-like test speeds!
